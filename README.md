# My_Components

这是我的组件库项目

# push request
核心格式如下：

```html
<类型>[可选作用域]: <描述>

[可选正文]

[可选脚注]
```

核心字段

| 字段       | 说明                                                         |
| ---------- | ------------------------------------------------------------ |
| 类型       | 必须，说明提交的核心目的（如 feat/fix/docs 等）              |
| 可选作用域 | 可选，指定修改的范围（如 button/table/utils 等，适配组件库） |
| 描述       | 必须，简短描述修改内容（首字母小写，不加句号，不超过 50 个字符） |
| 可选正文   | 可选，详细说明修改原因、方案、影响等                         |
| 可选脚注   | 可选，用于标注 Breaking Change（重大变更）、关联 Issue 等    |

组件库常用 Commit 类型

| 类型     | 适用场景                                                     |
| -------- | ------------------------------------------------------------ |
| feat     | 新增组件（如新增 Button 组件）、新增组件功能（如给 Table 加排序功能） |
| fix      | 修复组件 bug（如修复 Input 输入框防抖失效）                  |
| docs     | 修改文档（如更新 README、补充组件使用示例）                  |
| style    | 代码格式调整（如格式化代码、修改缩进，无逻辑变更）           |
| refactor | 代码重构（如优化组件内部逻辑，无新增 / 修复功能）            |
| test     | 添加 / 修改测试用例（如给 Button 组件加单元测试）            |
| chore    | 构建 / 工程配置变更（如修改 vite 配置、更新依赖、调整打包规则） |
| perf     | 性能优化（如优化列表渲染性能、减少组件重渲染）               |
| revert   | 回滚之前的提交（如 revert: feat (button): 新增禁用样式）     |

> 例子

### 二、适合你的组件库的 Commit 示例

#### 1. 新增组件

```
feat(button): 新增基础按钮组件

- 实现按钮的默认/主要/次要样式
- 支持禁用、加载状态
- 适配移动端尺寸
```

#### 2. 修复组件 bug

```
fix(table): 修复表格分页切换数据不刷新的问题

原因：分页参数未绑定到请求参数中
解决方案：将 pageNum/pageSize 同步到请求参数
```

#### 3. 更新文档

```
docs(readme): 补充组件库安装和快速上手说明
```

#### 4. 重构组件逻辑

```
refactor(input): 重构输入框防抖逻辑

- 抽离防抖函数到 utils 目录
- 优化防抖触发时机，减少无效请求
```

#### 5. 调整工程配置

```
chore: 更新 vite 到 7.3.1 版本

适配最新版 vite 的配置语法，修复打包警告
```

#### 6. 重大变更（Breaking Change）

```
feat(dialog): 重构弹窗组件 API

BREAKING CHANGE: 
- 移除 oldOpen 方法，统一为 open 方法
- props 中 visible 改为 v-model:visible 绑定
```
